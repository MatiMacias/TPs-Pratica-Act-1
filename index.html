Ventajas y desventajas de las herramientas de HTML5 de desarrollo de APPs para Android multiplataforma

Ventajas:

  • Permite desarrollar aplicaciones que se adaptan facilmente a distintas resoluciones, tama√±os de pantalla, relaciones de aspecto y orientaciones.
  • Permite potenciar las funcionalidades avanzadas, tales como el GPS, la camara y el acelerometro, en un dispositivo moderno y brindar una experiencia de usuario adaptada al contexto en una amplia variedad de dispositivos, incluidos los smartphones y las tabletas electronicas.
  • Las aplicaciones moviles pueden usar los mismos canales de monetizacion y distribucion que las aplicaciones nativas.
  • Las aplicaciones no estan restringidas por los marcos de las ventanas de los navegadores y se pueden ejecutar en modo de pantalla completa.
  • Los ususarios tienen el control completo de los dispositivos y el uso del espacio de pantalla.
  • El codigo centralizado se puede modificar para que se interconecte con varios dispositivos.

Desventajas:

  • A veces, lleva tiempo tener lista las aplicaciones moviles debido a que los desarrolladores tienen que escribir codigo para cada plataforma.
  • Los desarrolladores trabajan en lenguajes que no son nativos para las plataformas. por eso, la eficacia del codigo depende del motor de traduccion. sucede a menudo que las tecnicas de codificacion se ineficientes o que el codigo es redundante.
  • La diferencia de tiempos de ejecucion de las plataformas a menudo causa complicaciones en el desarrollo de aplicaciones moviles multiplataforma.
  • El estandar HTML5 evoluciono en un periodo relativamente corto y ello produjo discrepancias en la aplementacion de atributos CSS,etiquetas HTML5, y API de JavaScript. A veces, estas caracteristicas se comportan de manera diferentes en plataformas distintas. Sin embargo, hay herramientas disponibles para resolver estas discrepancias.
